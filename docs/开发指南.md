# 开发指南

## 快速启动

```bash
npm run dev
```

这一个命令会自动：
1. 启动本地存储服务（端口 `3001`）
2. 启动 Vite 开发服务器（端口 `5174`，热更新）
3. 按 `Ctrl+C` 同时关闭两个服务

启动后在浏览器打开 **http://localhost:5174**

## 所有脚本

| 命令 | 作用 | 影响数据？ |
|------|------|-----------|
| `npm run dev` | 一键启动：存储服务 + Vite | ❌ 不删数据 |
| `npm run storage` | 仅启动存储服务 | ❌ |
| `npm run dev:web` | 仅启动 Vite | ❌ |
| `npm run build` | 生产构建 | ❌ |
| `npm run preview` | 预览生产构建 | ❌ |
| `npm run backup` | 备份数据到 `data_backup_` | ❌ |
| `npm run wipe-data` | ⛔ 删除所有数据（三重确认） | ⚠️ 永久删除 |
| `npm run uninstall` | 卸载程序代码，保留数据 | ❌ 数据完整 |

## 服务说明

| 服务 | 端口 | 作用 | 数据 |
|------|------|------|------|
| `local-storage-server.mjs` | `3001` | 将前端数据持久化到本地文件系统 | `~/Documents/moyin-creator/data/` |
| Vite 开发服务器 | `5174` | 前端热更新 + API 代理 | 无 |

## 数据目录

```
~/Documents/moyin-creator/
├── data/                       ← 所有持久化数据
│   ├── _p/{projectId}/         ← 按项目隔离
│   ├── _shared/                ← 跨项目共享
│   ├── images/                 ← 生成的图片
│   └── _migrated.json          ← 迁移标记
└── data_backup_YYYYMMDD_HHMM/  ← 备份
```

详细设计见 [存储安全设计](./存储安全设计.md)。

## ⚠️ 安全启动规则

**永远不要**在存储服务未运行的情况下打开 WebUI 并操作数据。

- ✅ 用 `npm run dev` 一键启动（自动保证顺序）
- ✅ 数据目录 `~/Documents/moyin-creator/data/` 与程序代码物理隔离
- ✅ 删除程序代码不会影响数据
- ❌ 只开 Vite 不开存储服务 → 数据无法持久化，刷新即丢失
- ❌ Vite 已运行时再开一个 Vite → 端口冲突

## 生命周期操作

### 备份数据

```bash
npm run backup
```

自动创建 `~/Documents/moyin-creator/data_backup_日期时间/` 目录。

### 卸载程序（保留数据 ⭐）

```bash
npm run uninstall
```

- ✅ 删除所有程序代码（`~/Projects/moyin-creator-webui/`）
- ✅ 数据完好无损（`~/Documents/moyin-creator/data/`）
- ✅ 需要确认后才执行

### 重新安装

```bash
cd ~/Projects
git clone https://github.com/ilxu7z/moyin-creator-webui.git
cd moyin-creator-webui
npm install
npm run dev
```

之前的数据自动恢复，无需任何迁移操作。

### 删除所有数据 ⛔

```bash
npm run wipe-data
```

- ⚠️ 此操作不可逆
- ⚠️ 需要**三重确认**（输入中文短语 + yes）
- ⚠️ 删除前自动备份到 `data_deleted_日期时间/`
- 确认前会列出当前所有项目及文件数量

## 故障排查

| 症状 | 原因 | 解决 |
|------|------|------|
| `npm run dev` 报端口冲突 | 旧进程未退出 | `lsof -i :3001` / `lsof -i :5174` 找到 PID 后 `kill` |
| 前端操作后刷新数据消失 | 存储服务未启动 | 检查 `lsof -i :3001` 是否有 node 监听 |
| 图床上传失败 | CORS / 网络限制 | 默认使用 SCDN 图床；建议不要切换为海外图床 |
| 健康检查报告空项目目录 | 项目数据在此前 bug 中丢失 | 不可恢复，可删除空目录重建 |
| 重装后项目列表为空 | `discoverProjectsFromDisk` 未扫描到 `_p/` | 确认 `_p/` 下有包含 `.json` 文件的项目目录 |
| 构建报错 | tsc 类型检查不通过 | 先 `npx tsc --noEmit` 查看具体错误 |
