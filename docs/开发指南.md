# 开发指南

## 快速启动

```bash
npm run dev
```

这一个命令会自动：
1. 启动本地存储服务（端口 `3001`），数据保存在 `~/Documents/moyin-creator/data/`
2. 启动 Vite 开发服务器（端口 `5174`，热更新）
3. 按 `Ctrl+C` 同时关闭两个服务

启动后在浏览器打开 **http://localhost:5174**

## 单独启动

如果你只想启动存储服务或 Vite：

```bash
npm run storage          # 只启动存储服务（端口 3001）
npm run dev:web          # 只启动 Vite 开发服务器（端口 5174）
npm run build            # 生产构建 → dist/
npm run preview          # 预览生产构建（端口 5174）
```

## 服务说明

| 服务 | 端口 | 作用 | 数据 |
|------|------|------|------|
| `local-storage-server.mjs` | `3001` | 将前端数据持久化到本地文件系统 | `~/Documents/moyin-creator/data/` |
| Vite 开发服务器 | `5174` | 前端热更新 + API 代理 | 无 |

## ⚠️ 安全启动规则

**永远不要**在存储服务未运行的情况下打开 WebUI 并操作数据。

- ✅ 用 `npm run dev` 一键启动（自动保证顺序）
- ✅ 两个服务都启动后，前端可以正常读写项目数据
- ❌ 只开 Vite 不开存储服务 → 数据无法持久化，刷新即丢失
- ❌ Vite 已运行时再开一个 Vite → 端口冲突

## 数据备份

项目数据存储在 `~/Documents/moyin-creator/data/`。
所有持久化操作（Zustand persist）通过 `window.fileStorage` → HTTP API → 本地文件系统完成。

## 故障排查

| 症状 | 原因 | 解决 |
|------|------|------|
| `npm run dev` 报端口冲突 | 旧进程未退出 | `lsof -i :3001` / `lsof -i :5174` 找到 PID 后 `kill` |
| 前端操作后刷新数据消失 | 存储服务未启动 | 检查 `lsof -i :3001` 是否有 node 监听 |
| 图床上传失败 | CORS / 网络限制 | 默认使用 SCDN 图床；建议不要切换为海外图床 |
| 构建报错 | tsc 类型检查不通过 | 先 `npx tsc --noEmit` 查看具体错误 |
